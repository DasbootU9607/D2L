## 1. Perceptron

### 1.1 What a Perceptron Is

A perceptron is a simple linear classifier.

It receives an input vector:

$$
x = (x_1, x_2, ¥dots, x_d)
$$

It has a weight vector:

$$
w = (w_1, w_2, ¥dots, w_d)
$$

and a bias:

$$
b
$$

The perceptron computes a score:

$$
z = w^¥top x + b
$$

where:

$$
w^¥top x = w_1x_1 + w_2x_2 + ¥cdots + w_dx_d
$$

The model predicts by checking the sign of the score:

$$
¥hat{y} = ¥text{sign}(w^¥top x + b)
$$

Usually, perceptron labels are:

$$
y ¥in ¥{-1, +1¥}
$$

The rule is:

* If $w^¥top x + b > 0$, predict $+1$.
* If $w^¥top x + b ¥le 0$, predict $-1$.

### 1.2 Decision Boundary

The decision boundary is where the score equals zero:

$$
w^¥top x + b = 0
$$

In 2D, this is a line.

In higher dimensions, this is a hyperplane.

The weight vector $w$ controls the orientation of the boundary.

The bias $b$ shifts the boundary.

### 1.3 Correct Classification Condition

For a training example $(x_i, y_i)$, the perceptron classifies it correctly if:

$$
y_i(w^¥top x_i + b) > 0
$$

Why does this work?

If $y_i = +1$, then we want:

$$
w^¥top x_i + b > 0
$$

If $y_i = -1$, then we want:

$$
w^¥top x_i + b < 0
$$

Both cases are captured by:

$$
y_i(w^¥top x_i + b) > 0
$$

If:

$$
y_i(w^¥top x_i + b) ¥le 0
$$

then the example is either misclassified or lies exactly on the decision boundary.

### 1.4 Perceptron Training Algorithm

Initialize:

$$
w = 0,¥quad b = 0
$$

Then repeatedly scan the training data.

For each example $(x_i, y_i)$:

If:

$$
y_i(w^¥top x_i + b) ¥le 0
$$

update:

$$
w ¥leftarrow w + y_i x_i
$$

$$
b ¥leftarrow b + y_i
$$

Otherwise, do nothing.

### 1.5 Intuition Behind the Update Rule

If a positive example is misclassified, the score is too small.

So we move $w$ toward $x_i$:

$$
w ¥leftarrow w + x_i
$$

If a negative example is misclassified, the score is too large.

So we move $w$ away from $x_i$:

$$
w ¥leftarrow w - x_i
$$

The compact update rule is:

$$
w ¥leftarrow w + y_i x_i
$$

$$
b ¥leftarrow b + y_i
$$

### 1.6 Perceptron Loss

A common perceptron loss is:

$$
¥ell(y,x,w,b)=¥max(0,-y(w^¥top x+b))
$$

If the example is correctly classified:

$$
y(w^¥top x+b) > 0
$$

then:

$$
-y(w^¥top x+b) < 0
$$

so the loss is:

$$
0
$$

If the example is misclassified:

$$
y(w^¥top x+b) ¥le 0
$$

then the loss is positive.

This means the model needs to update its parameters.

### 1.7 Relationship to Gradient Descent

The perceptron algorithm can be viewed as a simple form of stochastic gradient descent.

Each update uses one example at a time.

When the example is already correctly classified, the loss is zero and no update is needed.

When the example is misclassified, the model updates $w$ and $b$ in the direction that makes the correct class more likely.

## 2. Single-Hidden-Layer Neural Network for Binary Classification

### 2.1 Model Structure

A single-hidden-layer neural network has three main parts:

1. Input layer
2. Hidden layer
3. Output layer

The input vector is:

$$
¥mathbf{x} ¥in ¥mathbb{R}^n
$$

where $n$ is the number of input features.

### 2.2 Hidden Layer

The hidden layer has $m$ hidden units.

Its weight matrix and bias vector are:

$$
¥mathbf{W}_1 ¥in ¥mathbb{R}^{m ¥times n}
$$

$$
¥mathbf{b}_1 ¥in ¥mathbb{R}^m
$$

The hidden representation is:

$$
¥mathbf{h} = ¥sigma(¥mathbf{W}_1 ¥mathbf{x} + ¥mathbf{b}_1)
$$

where:

* $¥mathbf{h} ¥in ¥mathbb{R}^m$
* $¥sigma$ is an element-wise activation function
* $¥mathbf{W}_1 ¥mathbf{x} + ¥mathbf{b}_1$ is a linear transformation
* $¥sigma(¥cdot)$ introduces non-linearity

Without an activation function, stacking layers would still only produce a linear model.

### 2.3 Output Layer

For binary classification, the output layer often produces one scalar:

$$
o = ¥mathbf{w}_2^T ¥mathbf{h} + b_2
$$

where:

$$
¥mathbf{w}_2 ¥in ¥mathbb{R}^m,¥quad b_2 ¥in ¥mathbb{R}
$$

The scalar output can be passed through a sigmoid function:

$$
¥hat{y} = ¥frac{1}{1 + e^{-o}}
$$

Then a threshold such as 0.5 can be used:

* If $¥hat{y} > 0.5$, predict class 1.
* Otherwise, predict class 0.

## 3. Activation Functions

Activation functions allow neural networks to learn nonlinear patterns.

### 3.1 Step Function

The step function is:

$$
f(x) =
¥begin{cases}
1, & x > 0 ¥¥
0, & ¥text{otherwise}
¥end{cases}
$$

Properties:

* Output range: $¥{0, 1¥}$
* Represents a hard decision
* Not differentiable at $x = 0$
* Not commonly used in modern neural networks because gradient-based optimization needs differentiable functions

### 3.2 Sigmoid Function

The sigmoid function is:

$$
¥sigma(x) = ¥frac{1}{1 + e^{-x}}
$$

Properties:

* Output range: $(0, 1)$
* Smooth and differentiable
* Often used to model probabilities
* Common in the output layer for binary classification

Its derivative is:

$$
¥sigma'(x) = ¥sigma(x)(1 - ¥sigma(x))
$$

The main limitation is that sigmoid can saturate when $x$ is very large or very small. In those regions, gradients become very small.

### 3.3 ReLU Function

The ReLU function is:

$$
¥text{ReLU}(x) = ¥max(0, x)
$$

Properties:

* Output is 0 for negative inputs.
* Output equals $x$ for positive inputs.
* Simple and efficient.
* Commonly used in hidden layers.

ReLU is important because it adds non-linearity while keeping optimization relatively simple.

## 4. Multi-Class Classification and Deep Neural Networks

### 4.1 Multi-Class Classification

In multi-class classification, the model predicts one class from $k$ possible classes.

The output is usually a vector:

$$
¥mathbf{y} = [y_1, y_2, ¥dots, y_k]
$$

where each $y_i$ represents the probability of class $i$.

### 4.2 Softmax Function

The model first produces raw scores called logits:

$$
¥mathbf{o} = [o_1, o_2, ¥dots, o_k]
$$

Softmax converts logits into probabilities:

$$
¥mathbf{y} = ¥text{softmax}(¥mathbf{o})
$$

For each class:

$$
y_i = ¥frac{e^{o_i}}{¥sum_{j=1}^{k} e^{o_j}}
$$

Softmax outputs satisfy:

$$
0 < y_i < 1
$$

and:

$$
¥sum_{i=1}^{k} y_i = 1
$$

The predicted class is:

$$
¥hat{c} = ¥arg¥max_i y_i
$$

### 4.3 One-Hidden-Layer Network for Multi-Class Classification

Input:

$$
¥mathbf{x} ¥in ¥mathbb{R}^n
$$

Hidden layer:

$$
¥mathbf{h} = ¥sigma(¥mathbf{W}_1 ¥mathbf{x} + ¥mathbf{b}_1)
$$

Output layer:

$$
¥mathbf{o} = ¥mathbf{W}_2^T ¥mathbf{h} + ¥mathbf{b}_2
$$

Final prediction:

$$
¥mathbf{y} = ¥text{softmax}(¥mathbf{o})
$$

### 4.4 Multi-Layer Neural Network

A deeper neural network stacks multiple hidden layers:

$$
¥mathbf{h}_1 = ¥sigma(¥mathbf{W}_1 ¥mathbf{x} + ¥mathbf{b}_1)
$$

$$
¥mathbf{h}_2 = ¥sigma(¥mathbf{W}_2 ¥mathbf{h}_1 + ¥mathbf{b}_2)
$$

$$
¥mathbf{h}_3 = ¥sigma(¥mathbf{W}_3 ¥mathbf{h}_2 + ¥mathbf{b}_3)
$$

The output layer produces logits:

$$
¥mathbf{o} = ¥mathbf{W}_4 ¥mathbf{h}_3 + ¥mathbf{b}_4
$$

For multi-class classification:

$$
¥mathbf{y} = ¥text{softmax}(¥mathbf{o})
$$

### 4.5 Parameters and Hyperparameters

Parameters are learned during training:

* Weight matrices such as $¥mathbf{W}_1, ¥mathbf{W}_2, ¥dots$
* Bias vectors such as $¥mathbf{b}_1, ¥mathbf{b}_2, ¥dots$

Hyperparameters are chosen before training:

* Number of hidden layers
* Number of hidden units
* Activation function
* Learning rate
* Batch size
* Number of epochs

## 5. MLP From Scratch

The notebook implements a multilayer perceptron for Fashion-MNIST.

Fashion-MNIST images are 28 by 28 pixels.

So each image has:

```text
28 * 28 = 784
```

input values.

### 5.1 Loading Data

```python
batch_size = 256
train_iter, test_iter = d2l.load_data_fashion_mnist(batch_size)
```

This loads the Fashion-MNIST dataset and groups examples into mini-batches of size 256.

### 5.2 Defining Parameters

```python
num_inputs, num_outputs, num_hiddens = 784, 10, 256
```

This means:

| Name | Meaning |
| --- | --- |
| `num_inputs = 784` | Each image is flattened into 784 input features |
| `num_outputs = 10` | There are 10 clothing classes |
| `num_hiddens = 256` | The hidden layer has 256 hidden units |

The parameters are:

```python
w1 = nn.Parameter(torch.randn(num_inputs, num_hiddens) * 0.01)
b1 = nn.Parameter(torch.zeros(num_hiddens))
w2 = nn.Parameter(torch.randn(num_hiddens, num_outputs) * 0.01)
b2 = nn.Parameter(torch.zeros(num_outputs))
```

Their shapes are:

| Parameter | Shape | Purpose |
| --- | --- | --- |
| `w1` | `(784, 256)` | Input to hidden layer |
| `b1` | `(256,)` | Hidden layer bias |
| `w2` | `(256, 10)` | Hidden to output layer |
| `b2` | `(10,)` | Output layer bias |

### 5.3 ReLU Implementation

```python
def relu(x):
    a = torch.zeros_like(x)
    return torch.max(x, a)
```

This implements:

$$
¥text{ReLU}(x) = ¥max(0, x)
$$

Negative values become 0, and positive values stay unchanged.

### 5.4 Model Definition

```python
def net(x):
    x = x.reshape((-1, num_inputs))
    h = relu(x @ w1 + b1)
    return h @ w2 + b2
```

Step by step:

1. `x.reshape((-1, num_inputs))` flattens each image into a vector of length 784.
2. `x @ w1 + b1` computes the hidden layer pre-activation.
3. `relu(...)` applies the nonlinear activation.
4. `h @ w2 + b2` computes output logits for 10 classes.

The output is not passed through softmax manually because `nn.CrossEntropyLoss` expects raw logits.

### 5.5 Loss and Training

```python
loss = nn.CrossEntropyLoss(reduction='none')
```

Cross-entropy loss is used for multi-class classification.

```python
num_epochs, lr = 10, 0.1
updater = torch.optim.SGD(params, lr=lr)
d2l.train_ch3(net, train_iter, test_iter, loss, num_epochs, updater)
```

This trains the model for 10 epochs using stochastic gradient descent.

## 6. Concise PyTorch Implementation

The notebook then implements the same MLP using PyTorch's high-level API.

### 6.1 Network Definition

```python
net = nn.Sequential(
    nn.Flatten(),
    nn.Linear(784, 256),
    nn.ReLU(),
    nn.Linear(256, 10)
)
```

Layer by layer:

| Layer | Purpose |
| --- | --- |
| `nn.Flatten()` | Converts each image into a 784-dimensional vector |
| `nn.Linear(784, 256)` | Input-to-hidden fully connected layer |
| `nn.ReLU()` | Nonlinear activation |
| `nn.Linear(256, 10)` | Hidden-to-output layer producing logits |

### 6.2 Weight Initialization

```python
def init_weights(m):
    if type(m) == nn.Linear:
        nn.init.normal_(m.weight, std=0.01)

net.apply(init_weights)
```

`net.apply(init_weights)` visits every layer in the network.

If the layer is `nn.Linear`, its weights are initialized from a normal distribution with standard deviation 0.01.

This is important because hidden units should not all start with identical weights.

### 6.3 Training

```python
batch_size, lr, num_epochs = 256, 0.1, 10
loss = nn.CrossEntropyLoss(reduction='none')
trainer = torch.optim.SGD(net.parameters(), lr=lr)

train_iter, test_iter = d2l.load_data_fashion_mnist(batch_size)
d2l.train_ch3(net, train_iter, test_iter, loss, num_epochs, trainer)
```

In this version, `net.parameters()` automatically collects all trainable parameters from the layers inside `nn.Sequential`.

So we do not need to manually create:

```python
params = [w1, b1, w2, b2]
```

### 6.4 Correspondence Between the Two Implementations

| From Scratch | High-Level API | Purpose |
| --- | --- | --- |
| `X.reshape((-1, 784))` | `nn.Flatten()` | Flatten each image |
| `W1`, `b1` | `nn.Linear(784, 256)` | Input-to-hidden layer |
| `relu(...)` | `nn.ReLU()` | Nonlinear activation |
| `W2`, `b2` | `nn.Linear(256, 10)` | Hidden-to-output layer |
| `params = [...]` | `net.parameters()` | Parameters passed to optimizer |

The from-scratch version is useful for understanding what the model computes internally.

The concise PyTorch version is closer to how neural networks are usually written in real projects.

## 7. What You Should Understand From This Notebook

### 7.1 Perceptron

A perceptron is a linear classifier.

It predicts based on:

$$
¥text{sign}(w^¥top x + b)
$$

It can only learn linear decision boundaries.

### 7.2 Hidden Layers

A hidden layer applies:

$$
¥mathbf{h} = ¥sigma(¥mathbf{W}¥mathbf{x} + ¥mathbf{b})
$$

The linear transformation extracts features, and the activation function introduces non-linearity.

### 7.3 MLP

A multilayer perceptron stacks linear layers and nonlinear activation functions.

For Fashion-MNIST, the model is:

```text
784 inputs -> 256 hidden units -> 10 output logits
```

### 7.4 Final Summary

The perceptron is the simplest linear classifier.

A multilayer perceptron extends this idea by adding hidden layers and nonlinear activation functions.

The key transition is:

```text
Perceptron: linear decision boundary
MLP: nonlinear decision boundary through hidden layers and activations
```
