# 8.Model Initialization Walkthrough

This walkthrough explains the notebook `8.Model Initialization.ipynb` step by step.

The notebook is about one practical question:

**Before training starts, how large should the initial weights be?**

If the weights are too large, activations and gradients may explode.

If the weights are too small, activations and gradients may vanish.

The notebook has four main ideas:

1. Initialization should keep signal scale stable.
2. Forward variance explains what happens to activations.
3. Backward variance explains what happens to gradients.
4. Xavier initialization chooses a weight scale that balances forward and backward stability.

---

## 1. Why Model Initialization Matters

### 1.1 What Initialization Means

Model initialization means choosing the starting values of the model parameters before training begins.

For a neural network layer, the most important parameters are usually the weights:

$$
\mathbf{W}^t
$$

The superscript $t$ means this is the weight matrix of layer $t$.

At the beginning of training, these weights are usually random.

But they should not be random with any arbitrary scale.

### 1.2 Why Scale Matters

In a deep network, numbers pass through many layers.

Forward pass:

$$
\text{input} \rightarrow \text{layer 1} \rightarrow \text{layer 2} \rightarrow \cdots \rightarrow \text{output}
$$

Backward pass:

$$
\text{loss gradient} \rightarrow \text{last layer} \rightarrow \cdots \rightarrow \text{first layer}
$$

Every layer multiplies values by weights.

If each layer makes the signal slightly larger, many layers can make it enormous.

If each layer makes the signal slightly smaller, many layers can make it almost zero.

This is why initialization matters.

### 1.3 The Practical Goal

The goal is not to find perfect initial weights.

The goal is simpler:

```text
Start from weights that let information flow through the network.
```

That means:

* activations should not explode or vanish in the forward pass;
* gradients should not explode or vanish in the backward pass.

## 2. Mean and Variance

### 2.1 Expected Value

The notation:

$$
\mathbb{E}[x]
$$

means the expected value of $x$.

You can think of it as the average value.

Example:

$$
[-1, 0, 1]
$$

has average:

$$
0.
$$

So its expected value is around zero.

### 2.2 Variance

The notation:

$$
\operatorname{Var}[x]
$$

means the variance of $x$.

Variance measures how spread out the values are.

For example:

$$
[-1, 0, 1]
$$

and

$$
[-100, 0, 100]
$$

both have mean zero.

But the second group is much more spread out, so it has much larger variance.

### 2.3 Why We Care About Variance

In neural networks, the exact value of every neuron is random at initialization.

Instead of tracking every number, we track the overall scale.

Variance is a convenient way to measure that scale.

The key question is:

```text
Does this layer make the variance bigger, smaller, or about the same?
```

---

## 3. Forward Variance

### 3.1 The Layer Formula

For one layer without activation:

$$
\mathbf{h}^t = \mathbf{W}^t\mathbf{h}^{t-1}
$$

where:

| Symbol | Meaning |
| --- | --- |
| $\mathbf{h}^{t-1}$ | Output from the previous layer |
| $\mathbf{W}^t$ | Weight matrix of the current layer |
| $\mathbf{h}^t$ | Output of the current layer |

If we look at one neuron in the current layer:

$$
h_i^t = \sum_{j=1}^{n_{t-1}} w_{ij}^t h_j^{t-1}
$$

This means:

```text
one current-layer output = many weighted previous-layer outputs added together
```

### 3.2 What $n_{t-1}$ Means

$n_{t-1}$ is the number of inputs to the current layer.

It is also called:

```text
fan_in
```

If $n_{t-1} = 100$, then each output neuron adds 100 terms.

If $n_{t-1} = 1000$, then each output neuron adds 1000 terms.

More added terms usually means the output can become larger.

### 3.3 Weight Assumptions

The derivation assumes:

$$
\mathbb{E}[w_{ij}^t] = 0
$$

and

$$
\operatorname{Var}[w_{ij}^t] = \gamma_t.
$$

Here $\gamma_t$ means the variance of the weights in layer $t$.

So:

```text
gamma_t = Var[w]
```

### 3.4 Forward Variance Result

The notebook derives:

$$
\operatorname{Var}[h_i^t]
=
n_{t-1}\gamma_t\operatorname{Var}[h_j^{t-1}].
$$

Read this as:

```text
current output variance
=
number of input terms
* weight variance
* previous output variance
```

### 3.5 Why We Want $n_{t-1}\gamma_t = 1$

We want the current layer to keep the variance about the same as the previous layer:

$$
\operatorname{Var}[h_i^t]
\approx
\operatorname{Var}[h_j^{t-1}].
$$

From the formula:

$$
\operatorname{Var}[h_i^t]
=
n_{t-1}\gamma_t\operatorname{Var}[h_j^{t-1}],
$$

the multiplier is:

$$
n_{t-1}\gamma_t.
$$

So we want:

$$
n_{t-1}\gamma_t = 1.
$$

That gives:

$$
\gamma_t = \frac{1}{n_{t-1}}.
$$

### 3.6 Forward Intuition

If a layer adds many input terms, each weight should be smaller.

Example:

If a neuron adds 100 terms, each term should be smaller than if the neuron adds only 10 terms.

Otherwise, the output scale will grow as the network gets deeper.

---

## 4. Backward Variance

### 4.1 What Backward Variance Tracks

Forward variance tracks activations.

Backward variance tracks gradients.

The gradient tells the model how to update its parameters.

If gradients become too small, early layers learn very slowly.

If gradients become too large, updates become unstable.

### 4.2 Backward Formula

Ignoring activation again:

$$
\frac{\partial \ell}{\partial \mathbf{h}^{t-1}}
=
(\mathbf{W}^t)^T
\frac{\partial \ell}{\partial \mathbf{h}^t}.
$$

This means:

```text
gradient sent to previous layer
=
current layer gradient multiplied by the weight matrix
```

The transpose appears because the gradient is moving in the opposite direction of the forward pass.

### 4.3 What $n_t$ Means

$n_t$ is the number of outputs of the current layer.

It is also called:

```text
fan_out
```

During the backward pass, one previous-layer neuron receives gradient contributions from the current layer's output neurons.

So the number of terms is related to $n_t$.

### 4.4 Backward Variance Result

The notebook gives:

$$
\operatorname{Var}\left[
\frac{\partial \ell}{\partial h_i^{t-1}}
\right]
=
n_t\gamma_t
\operatorname{Var}\left[
\frac{\partial \ell}{\partial h_j^t}
\right].
$$

Read this as:

```text
previous-layer gradient variance
=
number of output terms
* weight variance
* current-layer gradient variance
```

### 4.5 Why We Want $n_t\gamma_t = 1$

We want the gradient variance to stay about the same while moving backward:

$$
\operatorname{Var}\left[
\frac{\partial \ell}{\partial h_i^{t-1}}
\right]
\approx
\operatorname{Var}\left[
\frac{\partial \ell}{\partial h_j^t}
\right].
$$

From the formula, the multiplier is:

$$
n_t\gamma_t.
$$

So we want:

$$
n_t\gamma_t = 1.
$$

That gives:

$$
\gamma_t = \frac{1}{n_t}.
$$

### 4.6 Backward Intuition

If many current-layer neurons send gradient back to one previous-layer neuron, the sum can become large.

To prevent this, the weights should be smaller when `fan_out` is larger.

---

## 5. The Conflict Between Forward and Backward Requirements

Forward stability wants:

$$
\gamma_t = \frac{1}{n_{t-1}}.
$$

Backward stability wants:

$$
\gamma_t = \frac{1}{n_t}.
$$

But usually:

$$
n_{t-1} \ne n_t.
$$

So one weight variance cannot satisfy both conditions exactly.

This is the reason Xavier initialization exists.

It chooses a compromise.

---

## 6. Xavier Initialization

### 6.1 What Xavier Tries to Do

Xavier initialization tries to keep both of these stable:

1. Forward activations
2. Backward gradients

It does this by choosing the weight variance based on both:

| Name | Meaning |
| --- | --- |
| `fan_in` | Number of inputs to the layer |
| `fan_out` | Number of outputs from the layer |

In the notebook notation:

$$
\text{fan\_in} = n_{t-1}
$$

and

$$
\text{fan\_out} = n_t.
$$

### 6.2 The Xavier Formula

Xavier uses:

$$
\gamma_t = \frac{2}{n_{t-1}+n_t}.
$$

Equivalently:

$$
\operatorname{Var}[w]
=
\frac{2}{\text{fan\_in}+\text{fan\_out}}.
$$

This is the main formula.

### 6.3 Why There Is a 2 on Top

The forward condition wants:

$$
\gamma_t = \frac{1}{n_{t-1}}.
$$

The backward condition wants:

$$
\gamma_t = \frac{1}{n_t}.
$$

Xavier balances the two by using the average size of `fan_in` and `fan_out`.

The average is:

$$
\frac{n_{t-1}+n_t}{2}.
$$

So we choose:

$$
\gamma_t
=
\frac{1}{(n_{t-1}+n_t)/2}.
$$

Dividing by a fraction is the same as multiplying by its reciprocal:

$$
\gamma_t
=
\frac{2}{n_{t-1}+n_t}.
$$

That is where the 2 comes from.

### 6.4 Concrete Example

Suppose a layer has 100 inputs and 50 outputs.

Then:

$$
\text{fan\_in} = 100
$$

and

$$
\text{fan\_out} = 50.
$$

Xavier variance is:

$$
\operatorname{Var}[w]
=
\frac{2}{100+50}
=
\frac{2}{150}
\approx 0.0133.
$$

If we use a normal distribution, the standard deviation is:

$$
\sqrt{0.0133}
\approx
0.115.
$$

So the weights can be sampled like:

$$
w \sim \mathcal{N}(0, 0.0133).
$$

Or equivalently:

```text
mean = 0
standard deviation = about 0.115
```

### 6.5 Xavier Normal

For a normal distribution:

$$
w \sim \mathcal{N}
\left(
0,
\frac{2}{n_{t-1}+n_t}
\right).
$$

Be careful:

In this notation, the second value is the variance.

The standard deviation is:

$$
\sqrt{\frac{2}{n_{t-1}+n_t}}.
$$

Deep learning libraries usually ask for standard deviation, not variance.

### 6.6 Xavier Uniform

For a uniform distribution:

$$
w \sim \mathcal{U}[-a,a].
$$

A uniform distribution from $-a$ to $a$ has variance:

$$
\operatorname{Var}(\mathcal{U}[-a,a])
=
\frac{a^2}{3}.
$$

Xavier wants this variance:

$$
\frac{2}{n_{t-1}+n_t}.
$$

So set:

$$
\frac{a^2}{3}
=
\frac{2}{n_{t-1}+n_t}.
$$

Multiply both sides by 3:

$$
a^2
=
\frac{6}{n_{t-1}+n_t}.
$$

Take the square root:

$$
a
=
\sqrt{
\frac{6}{n_{t-1}+n_t}
}.
$$

Therefore:

$$
w \sim
\mathcal{U}
\left(
-\sqrt{\frac{6}{n_{t-1}+n_t}},
\sqrt{\frac{6}{n_{t-1}+n_t}}
\right).
$$

### 6.7 What to Remember About Xavier

You do not need to memorize every derivation first.

The important idea is:

```text
Xavier chooses the random weight scale from the layer shape.
```

More inputs or outputs means each weight should usually start smaller.

## 7. Activation Functions and Initialization

### 7.1 What the Previous Derivation Ignored

So far, we used:

$$
\mathbf{h}^t = \mathbf{W}^t\mathbf{h}^{t-1}.
$$

But a real neural network layer usually does this:

$$
\mathbf{h}' = \mathbf{W}^t\mathbf{h}^{t-1}
$$

then:

$$
\mathbf{h}^t = \sigma(\mathbf{h}').
$$

Here:

| Symbol | Meaning |
| --- | --- |
| $\mathbf{h}'$ | Value before activation |
| $\sigma$ | Activation function |
| $\mathbf{h}^t$ | Value after activation |

So even if the weight matrix keeps variance stable, the activation function can still change the scale.

### 7.2 The Ideal Activation Near Zero

At initialization, many values are close to zero.

So we ask:

```text
What does the activation function look like near zero?
```

The ideal case is:

$$
\sigma(x) \approx x.
$$

This means:

```text
input goes in, almost the same value comes out
```

That keeps the signal scale stable.

### 7.3 Linear Approximation

Near zero, we can approximate many functions by a line:

$$
\sigma(x) \approx \alpha x + \beta.
$$

Here:

| Symbol | Meaning |
| --- | --- |
| $\alpha$ | Slope |
| $\beta$ | Vertical shift |

For initialization, the best simple case is:

$$
\alpha = 1
$$

and:

$$
\beta = 0.
$$

Why?

If $\alpha = 1$, the function does not shrink or enlarge values near zero.

If $\beta = 0$, the function does not shift the mean away from zero.

So:

$$
\sigma(x) \approx x
$$

is the stable case.

### 7.4 Sigmoid Near Zero

Sigmoid is:

$$
\operatorname{sigmoid}(x)
=
\frac{1}{1+e^{-x}}.
$$

Near zero, it behaves like:

$$
\operatorname{sigmoid}(x)
\approx
\frac{1}{2}
+
\frac{x}{4}.
$$

Compare this with:

$$
\alpha x + \beta.
$$

For sigmoid:

$$
\alpha = \frac{1}{4}
$$

and:

$$
\beta = \frac{1}{2}.
$$

This is not ideal.

Two problems:

* $\beta = 1/2$ means the output is not centered at zero.
* $\alpha = 1/4$ means the signal is shrunk near zero.

This is one reason sigmoid can make deep networks harder to train.

### 7.5 Tanh Near Zero

Tanh behaves like:

$$
\tanh(x)
\approx
x
$$

near zero.

So for tanh:

$$
\alpha \approx 1
$$

and:

$$
\beta \approx 0.
$$

This is much closer to the ideal case.

That is why Xavier initialization is often associated with tanh-like activations.

### 7.6 ReLU Near Zero

ReLU is:

$$
\operatorname{ReLU}(x) = \max(0,x).
$$

If $x > 0$, then:

$$
\operatorname{ReLU}(x) = x.
$$

If $x < 0$, then:

$$
\operatorname{ReLU}(x) = 0.
$$

So ReLU keeps positive values but removes negative values.

That changes the variance.

Because ReLU throws away roughly half of a zero-centered signal, ReLU usually uses Kaiming initialization instead of Xavier initialization.

### 7.7 Adjusted Sigmoid

The notebook mentions:

$$
4\operatorname{sigmoid}(x)-2.
$$

Why does this help?

Start with the near-zero approximation:

$$
\operatorname{sigmoid}(x)
\approx
\frac{1}{2}+\frac{x}{4}.
$$

Multiply by 4:

$$
4\operatorname{sigmoid}(x)
\approx
2+x.
$$

Subtract 2:

$$
4\operatorname{sigmoid}(x)-2
\approx
x.
$$

So the adjusted sigmoid behaves more like the identity function near zero.

That means:

```text
it is more centered and has a better slope near zero
```

### 7.8 What to Remember About Activations

Initialization and activation functions are connected.

The weight initialization controls the scale before the activation.

The activation function then changes the scale again.

So a good initialization method depends on the activation function.

Simple rule:

| Activation | Common initialization idea |
| --- | --- |
| tanh | Xavier |
| sigmoid | Xavier may help, but sigmoid itself can still cause vanishing gradients |
| ReLU | Kaiming is usually better |

## 8. Simple Explanation of the Whole Lecture

### 8.1 The Network as a Chain of Multiplications

A deep network is like a long chain.

Each layer does something like:

$$
\text{new signal}
=
\text{weight matrix}
\times
\text{old signal}.
$$

If the weight scale is wrong, every layer makes the problem worse.

### 8.2 If Weights Are Too Large

Suppose every layer multiplies the signal scale by 1.2.

After 10 layers:

$$
1.2^{10} \approx 6.19.
$$

After 50 layers:

$$
1.2^{50} \approx 9100.
$$

So even a small amplification per layer becomes huge in a deep network.

This causes exploding activations or exploding gradients.

### 8.3 If Weights Are Too Small

Suppose every layer multiplies the signal scale by 0.8.

After 10 layers:

$$
0.8^{10} \approx 0.107.
$$

After 50 layers:

$$
0.8^{50} \approx 0.000014.
$$

So even a small shrinkage per layer can make the signal almost disappear.

This causes vanishing activations or vanishing gradients.

### 8.4 What Xavier Does

Xavier says:

```text
Do not choose one fixed random scale for every layer.
Choose the scale based on the layer's fan_in and fan_out.
```

The formula is:

$$
\operatorname{Var}[w]
=
\frac{2}{\text{fan\_in}+\text{fan\_out}}.
$$

This keeps the signal size more stable at the beginning of training.

### 8.5 How to Read the Formula

If `fan_in` and `fan_out` are large:

$$
\text{fan\_in}+\text{fan\_out}
$$

is large.

Then:

$$
\operatorname{Var}[w]
$$

is small.

So the weights start smaller.

If the layer has fewer inputs and outputs, the weights can start larger.

This makes sense because fewer terms are being added together.

### 8.6 One Mental Model

Think of each layer as a mixing bowl.

It mixes many incoming numbers together.

If there are many ingredients going into the bowl, each ingredient should be measured more carefully.

In neural network terms:

```text
more inputs/outputs -> smaller initial weights
```

That is the intuition behind Xavier.

## 9. Common Confusions

### 9.1 Is Xavier Trying to Find the Best Weights?

No.

Xavier does not find the final trained weights.

It only chooses a reasonable starting scale.

Training still has to learn the actual useful weights through gradient descent.

### 9.2 Why Do We Use Random Weights?

If all weights started with the same value, many neurons would learn the same thing.

Random initialization breaks this symmetry.

But the random values still need the right scale.

So the goal is:

```text
random, but not too large or too small
```

### 9.3 Why Not Always Use $\mathcal{N}(0, 0.01^2)$?

A fixed scale like 0.01 ignores the layer shape.

For a small layer, it may be fine.

For a very wide or deep network, it may be too small or too large.

Xavier adapts to the layer dimensions.

### 9.4 Why Does Activation Matter If Xavier Already Fixes the Weight Scale?

Because the layer is not only weights.

The layer is usually:

$$
\text{linear transformation}
+
\text{activation function}.
$$

The activation function can shrink, shift, or cut off the signal.

So initialization and activation must be considered together.
