# 6.Models Training Walkthrough

This walkthrough explains the notebook `6.Models Training.ipynb` step by step.

The main idea of this notebook is:

**A model is not better simply because it is more complex or because it performs well on the training data. What we really care about is whether it performs well on new data.**

The notebook has three major parts:

1. Model selection, underfitting, and overfitting
2. Weight decay as a regularization method
3. Dropout as another regularization method

---

## 1. Training Error and Generalization Error

### 1.1 Training Error

Training error is:

**The model's error on the training data.**

For example, if a model practices on 100 questions and is then tested on the same 100 questions, the error on those questions is the training error.

A low training error does not always mean the model has learned the real pattern. It may have only memorized the training examples.

### 1.2 Generalization Error

Generalization error is:

**The model's error on new data that it has not seen before.**

This is the error we actually care about.

If a student can solve all practice questions but fails when the exam questions change, the student has low training error but high generalization error. Machine learning models can have the same problem.

### 1.3 Validation Set and Test Set

Datasets are usually split into several parts:

| Dataset | Purpose |
| --- | --- |
| Training set | Used to learn model parameters |
| Validation set | Used to tune hyperparameters and select models |
| Test set | Used only at the end to estimate final performance |

The test set should not be used repeatedly to tune the model.

If you repeatedly adjust the model based on test results, the test set is no longer a fair simulation of unseen data.

### 1.4 K-Fold Cross-Validation

When the dataset is small, K-fold cross-validation can make evaluation more stable.

The procedure is:

1. Split the data into K parts.
2. Use one part as the validation set.
3. Use the remaining K - 1 parts as the training set.
4. Repeat this K times.
5. Report the average validation error.

Common choices are K = 5 or K = 10.

The simple intuition is:

**When data is limited, let each part of the data take a turn as the validation set.**

## 2. Underfitting and Overfitting

### 2.1 Underfitting

Underfitting means:

**The model is too simple to learn even the training data well.**

For example, if the true pattern is a curved polynomial but we use only a straight line, the model will not fit the data well.

Typical behavior:

| Training Error | Test Error |
| --- | --- |
| High | High |

The model performs poorly on both training data and new data.

### 2.2 Overfitting

Overfitting means:

**The model is too complex and memorizes noise in the training data.**

For example, if the true pattern is a smooth curve but the model bends aggressively to pass through every training point, it may learn random noise instead of the real pattern.

Typical behavior:

| Training Error | Test Error |
| --- | --- |
| Very low | High |

The model looks strong on the training data but fails on new data.

### 2.3 Model Capacity

Model capacity means:

**How complex a pattern the model can represent.**

Low-capacity models:

* Have weak expressive power
* Are more likely to underfit
* Example: using a straight line to fit a complex curve

High-capacity models:

* Have strong expressive power
* Are more likely to overfit
* Example: a high-degree polynomial can bend enough to memorize training points

### 2.4 Matching Model Capacity and Data Complexity

The notebook uses this idea:

| Model Capacity \ Data | Simple Data | Complex Data |
| --- | --- | --- |
| Low capacity | Normal | Underfitting |
| High capacity | Overfitting | Normal |

The key point is:

```text
The model complexity should match the data complexity.
```

Data complexity depends on factors such as:

* Number of examples
* Number of features in each example
* Temporal or spatial structure
* Diversity of the data

### 2.5 VC Dimension

The notebook also mentions VC dimension.

You can understand VC dimension as:

**A theoretical way to measure model capacity.**

For a classification model, the VC dimension roughly describes the largest number of points that the model can classify perfectly under any possible labeling.

For example, a perceptron with 2-dimensional inputs has VC dimension 3.

For a perceptron with N-dimensional inputs, the VC dimension is N + 1.

In deep learning, VC dimension is rarely used directly because:

* It is hard to compute for deep models.
* It often does not explain modern deep learning generalization very accurately.

So for now, the important takeaway is:

```text
VC dimension is a theoretical tool for describing model capacity, but it is not usually used directly when training deep learning models.
```

## 3. Polynomial Regression Experiment

The first experiment uses polynomial fitting to show underfitting and overfitting.

### 3.1 Importing Libraries

```python
import math
import numpy as np
import torch
from torch import nn
from d2l import torch as d2l
```

These libraries are used for:

| Library | Purpose |
| --- | --- |
| `math` | Factorials through the gamma function |
| `numpy` | Data generation and array operations |
| `torch` | Tensor computation |
| `nn` | PyTorch neural network modules |
| `d2l` | Helper functions from Dive into Deep Learning |

### 3.2 Generating Synthetic Data

```python
max_degree = 20
n_train, n_test = 100, 100
true_w = np.zeros(max_degree)
true_w[0:4] = np.array([5, 1.2, -3.4, 5.6])
```

This means:

* We prepare polynomial features up to 20 terms.
* We use 100 training examples.
* We use 100 test examples.
* The real data-generating pattern only uses the first 4 terms.

The true function is:

```text
y = 5 + 1.2x - 3.4 * x^2 / 2! + 5.6 * x^3 / 3! + noise
```

So the true model is a third-degree polynomial.

Why does a third-degree polynomial have 4 coefficients?

| Position | Meaning |
| --- | --- |
| Term 0 | Constant term |
| Term 1 | x |
| Term 2 | x^2 |
| Term 3 | x^3 |

The slice `true_w[0:4]` uses indices 0, 1, 2, and 3. It does not include index 4.

### 3.3 Building Polynomial Features

```python
poly_features = np.power(features, np.arange(max_degree).reshape(1, -1))
```

Each original input `x` is transformed into:

```text
[1, x, x^2, x^3, x^4, ..., x^19]
```

Because `max_degree = 20`, each example has 20 polynomial features.

If `x = 2`, the features look like:

```text
[1, 2, 4, 8, 16, ..., 2^19]
```

### 3.4 Dividing by Factorials

```python
for i in range(max_degree):
    poly_features[:, i] /= math.gamma(i + 1)
```

`math.gamma(i + 1)` equals `i!`.

So the features become:

```text
[1, x, x^2 / 2!, x^3 / 3!, x^4 / 4!, ...]
```

This keeps high-degree features from becoming too large, which makes training more stable.

### 3.5 Generating Labels

```python
labels = np.dot(poly_features, true_w)
labels += np.random.normal(scale=0.1, size=labels.shape)
```

The first line computes labels using the true polynomial.

The second line adds random noise.

Noise is added because real-world data is rarely perfectly clean.

## 4. Training the Polynomial Model

### 4.1 Converting Data to Tensors

```python
true_w, features, poly_features, labels = [
    torch.tensor(x, dtype=torch.float32)
    for x in [true_w, features, poly_features, labels]
]
```

The data is generated with NumPy, but the model is trained with PyTorch. Therefore, the arrays are converted into PyTorch tensors.

### 4.2 Evaluating Loss

```python
def evaluate_loss(net, data_iter, loss):
    metric = d2l.Accumulator(2)
    for X, y in data_iter:
        out = net(X)
        y = y.reshape(out.shape)
        l = loss(out, y)
        metric.add(l.sum(), l.numel())
    return metric[0] / metric[1]
```

This function computes the average loss of a model on a dataset.

The accumulator stores:

1. Total loss
2. Number of examples

Finally, it returns:

```text
average loss = total loss / number of examples
```

### 4.3 Why a Linear Layer Can Fit a Polynomial

The model is:

```python
net = nn.Sequential(nn.Linear(input_shape, 1, bias=False))
```

This is a linear layer, but the input is not only the original `x`.

If the input is:

```text
[1, x, x^2, x^3]
```

then the linear layer computes:

```text
w0 * 1 + w1 * x + w2 * x^2 + w3 * x^3
```

That is a polynomial.

So:

```text
Linear layer + polynomial features = polynomial regression
```

### 4.4 Why `bias=False`

The input already contains the constant feature `1`.

That means:

```text
w0 * 1
```

already acts like a bias term.

Adding another bias would create a redundant constant term.

### 4.5 Mini-Batch Training

```python
batch_size = min(10, train_labels.shape[0])
```

The model trains on small batches of examples instead of using all data at once.

This is mini-batch stochastic gradient descent.

The "stochastic" part means that each update uses a small batch of data, usually after shuffling the dataset.

## 6. Weight Decay

The problem from the previous experiment is:

**A model with too much capacity can overfit.**

One solution is to prevent the model weights from becoming too large.

This is weight decay.

### 6.1 What Weight Decay Does

Without weight decay, training only minimizes prediction error:

```text
loss
```

With weight decay, training minimizes:

```text
loss + lambda * weight penalty
```

The weight penalty is usually:

```text
1/2 * ||w||^2
```

This is the L2 penalty.

The intuition is:

**If the model tries to memorize the training data using very large weights, we penalize it.**

### 6.2 Hard Constraint and Soft Constraint

A hard constraint looks like:

```text
minimize loss(w, b)
subject to ||w||^2 <= theta
```

This directly says the weights cannot exceed a certain size.

A soft constraint looks like:

```text
minimize loss(w, b) + lambda / 2 * ||w||^2
```

This does not strictly forbid large weights, but large weights increase the loss.

In practice, the soft constraint is easier to optimize and is more commonly used.

### 6.3 Why It Is Called Weight Decay

With L2 regularization, the weight update has the form:

```text
w_{t+1} = (1 - eta * lambda) * w_t - eta * original_gradient
```

The factor:

```text
(1 - eta * lambda)
```

is usually smaller than 1.

So each update slightly shrinks the old weight before applying the gradient update.

That is why it is called:

```text
weight decay
```

## 7. Weight Decay Code

### 7.1 Why This Dataset Overfits Easily

```python
n_train, n_test, num_inputs, batch_size = 20, 100, 200, 5
```

This setup is intentional:

* Only 20 training examples
* 200 input features

This means:

```text
There are many parameters but very little training data.
```

This situation is very likely to overfit.

### 7.2 Initializing Parameters

```python
def init_params():
    w = torch.normal(0, 1, size=(num_inputs, 1), requires_grad=True)
    b = torch.zeros(1, requires_grad=True)
    return [w, b]
```

`requires_grad=True` tells PyTorch to compute gradients for these parameters.

### 7.3 L2 Penalty

```python
def l2_penalty(w):
    return torch.sum(w.pow(2)) / 2
```

This computes:

```text
1/2 * ||w||^2
```

The division by 2 makes the derivative cleaner:

```text
derivative of (1/2) * w^2 = w
```

### 7.4 Training From Scratch

The key line is:

```python
l = loss(net(X), y) + lambd * l2_penalty(w)
```

Without weight decay:

```text
loss = prediction error
```

With weight decay:

```text
loss = prediction error + weight penalty
```

The model is now encouraged to:

1. Predict accurately.
2. Keep the weights small.

### 7.5 Why Penalize `w` but Not `b`

Weight decay usually penalizes the weight vector `w`, not the bias `b`.

The weights control how strongly each input feature affects the output.

Large weights can make the model overly sensitive to specific features.

The bias mostly shifts the output up or down, so it is usually not treated as the main source of model complexity.

### 7.6 Comparing `lambd=0` and `lambd=3`

When `lambd=0`:

```text
loss = prediction error
```

There is no regularization. The model can easily overfit.

When `lambd=3`:

```text
loss = prediction error + 3 * L2 penalty
```

The model is pushed toward smaller weights.

Expected behavior:

| Setting | Training Loss | Test Loss | L2 Norm of w |
| --- | --- | --- | --- |
| `lambd=0` | Lower | Higher | Larger |
| `lambd=3` | Slightly higher | Lower | Smaller |

### 7.7 Concise PyTorch Implementation

PyTorch can apply weight decay directly in the optimizer:

```python
trainer = torch.optim.SGD([
    {"params": net[0].weight, "weight_decay": wd},
    {"params": net[0].bias}
], lr=lr)
```

This creates two parameter groups:

| Parameter | Weight Decay? |
| --- | --- |
| `net[0].weight` | Yes |
| `net[0].bias` | No |

The optimizer handles the L2 penalty internally.

## 8. Dropout

Dropout is another regularization method.

Its core idea is:

**During training, randomly set some hidden units' outputs to zero.**

For example, if a layer output is:

```text
[2.0, 1.5, 0.7, 3.2]
```

dropout may randomly change it to:

```text
[2.0, 0, 0.7, 0]
```

On another training step, it may become:

```text
[0, 1.5, 0.7, 3.2]
```

This prevents the model from relying too heavily on a small number of hidden units.

### 8.1 Why Dropout Is Regularization

Regularization tries to reduce overfitting.

Weight decay does this by:

```text
penalizing large weights
```

Dropout does this by:

```text
adding random noise during training
```

Both methods make the model less likely to memorize the training set.

### 8.2 Dropout Formula

The notebook uses:

```text
x_i' = 0                 with probability p
x_i' = x_i / (1 - p)     otherwise
```

Here `p` is the dropout probability.

If `p = 0.5`, each element has a 50% chance of being dropped.

Why divide by `1 - p`?

Because we want the expected value to stay the same.

Example:

```text
x = 10
p = 0.5
```

Then:

* 50% chance: output 0
* 50% chance: output 10 / 0.5 = 20

The expected output is:

```text
0.5 * 0 + 0.5 * 20 = 10
```

So dropout adds noise without changing the average scale.

### 8.3 The `dropout_layer` Function

```python
def dropout_layer(X, dropout):
    assert 0 <= dropout <= 1
    if dropout == 1:
        return torch.zeros_like(X)
    if dropout == 0:
        return X
    mask = (torch.randn(X.shape) > dropout).float()
    return mask * X / (1.0 - dropout)
```

The mask contains:

```text
1 = keep the element
0 = drop the element
```

Then:

```python
mask * X / (1.0 - dropout)
```

drops some elements and rescales the kept elements.

### 8.4 Dropout During Training and Inference

Dropout is used only during training.

During training:

```text
randomly drop hidden unit outputs
```

During inference:

```text
use the full network without random dropping
```

This is because predictions should be deterministic. The same input should produce the same output.

PyTorch's `nn.Dropout` follows this behavior:

* `net.train()` enables dropout.
* `net.eval()` disables dropout.

### 8.5 Dropout Network in the Notebook

The notebook defines an MLP for Fashion-MNIST:

```text
Input: 784
Hidden layer 1: 256
Hidden layer 2: 256
Output: 10
```

Dropout is added after each hidden layer:

```text
Linear -> ReLU -> Dropout -> Linear -> ReLU -> Dropout -> Linear
```

The concise implementation uses:

```python
nn.Dropout(dropout1)
nn.Dropout(dropout2)
```

This is the standard PyTorch way to use dropout.

## 9. Weight Decay vs. Dropout

| Method | What It Does | Main Effect |
| --- | --- | --- |
| Weight decay | Penalizes large weights | Makes parameters smaller and smoother |
| Dropout | Randomly drops hidden unit outputs during training | Prevents dependence on a few hidden units |

Both are regularization methods.

A simple way to remember the difference:

```text
Weight decay regularizes the parameter values.
Dropout regularizes the training process by adding random perturbations.
```

## 10. What You Should Understand From This Notebook

### 10.1 Training Error and Generalization Error Are Different

Low training error does not guarantee a good model.

The real goal is good performance on unseen data.

### 10.2 Underfitting

The model is too simple.

Typical behavior:

* Training error is high.
* Test error is high.

Possible solutions:

* Increase model complexity.
* Add useful features.
* Train longer.

### 10.3 Overfitting

The model is too complex relative to the data.

Typical behavior:

* Training error is low.
* Test error is high.

Possible solutions:

* Add more data.
* Reduce model complexity.
* Use regularization.
* Use weight decay or dropout.

### 10.4 Final Summary

This notebook is not mainly about memorizing formulas.

It teaches an important modeling judgment:

**A model that is too simple underfits, and a model that is too complex overfits. We use validation sets, test sets, and regularization methods such as weight decay and dropout to find models that perform better on new data.**
